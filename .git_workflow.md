# Developer Guidelines — Git & GitHub Workflow for Antigravity Agents

This file outlines the required Git and GitHub development workflow for all agentic coding sessions inside this workspace. All AI agents MUST follow these instructions proactively.

## 🛠️ Git Environment & Location
- Git is installed on this host at the absolute path: `C:\Program Files\Git\cmd\git.exe`
- Always call `git` commands using this absolute path: `& "C:\Program Files\Git\cmd\git.exe" <command>`

## 🔄 Development Flow Rules
1. **Status Checks**: At the beginning of a session or a major feature implementation, run `git status` to get a baseline of all changes.
2. **Incremental Commits**: Commit changes locally to the `master` branch whenever a milestone is reached or a critical bug is fixed. Do not lump unrelated features into a single massive commit.
3. **Descriptive Commit Messages**: Write professional, human-readable commit messages summarizing the changes (e.g., `git commit -m "Implement settings tab, exotics exclusions, table columns, and scroll fixes"`).
4. **Push Proactively**: Once a task or feature has been fully completed and verified, run `git push origin master` to sync changes with the remote repository on GitHub so it can be deployed instantly.
5. **No Workarounds**: Never use manual file upload workarounds if Git is available. Always verify environment paths and run Git natively.
