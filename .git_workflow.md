# Developer Guidelines — Git & GitHub Workflow for Antigravity Agents

This file outlines the required Git and GitHub development workflow for all agentic coding sessions inside this workspace. All AI agents MUST follow these instructions proactively.

## 🔄 Development Flow Rules
1. **Global Git Usage**: Git is globally registered in the system PATH. Always call standard, native Git commands directly in the shell (e.g. `git status`, `git commit`, `git push`).
2. **Incremental Commits**: Commit changes locally to the tracking branch whenever a milestone is reached or a critical bug is fixed. Do not lump unrelated features into a single massive commit.
3. **Descriptive Commit Messages**: Write professional, human-readable commit messages summarizing the changes (e.g., `git commit -m "Implement settings tab and exotic exclusions"`).
4. **Push Proactively**: Once a task or feature has been fully completed and verified, run `git push` to sync changes with the remote repository on GitHub so they can be deployed instantly.
